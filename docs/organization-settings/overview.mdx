# Organization Settings

Organization Settings is where you configure how Levelset works for your organization. Changes here apply to all locations within your organization.

## Accessing Settings

1. Click the profile icon in the top-right corner of the navigation
2. Select **Organization Settings** from the dropdown

### Users and Security

Manage who can access Levelset:

- **Users** - Add and manage dashboard users
- **Roles** - Define and customize the role hierarchy
- **Permissions** - Configure role-based access (coming soon)

### Mobile App

Configure the mobile PWA experience:

- **Access** - QR codes and links for mobile access
- **Configuration** - Mobile app settings (coming soon)

### General

Organization and location details:

- **Location Details** - Logo and location-specific information
- **Organization Details** - Organization-wide settings

## Who Can Access Settings

Organization Settings requires appropriate permissions:

| Setting Area | Minimum Role |
|--------------|--------------|
| Positional Excellence | Executive/Director |
| Discipline | Executive/Director |
| Users | Operator |
| Roles | Operator |
| General | Executive/Director |

<Note>
Levelset Admins have full access to all settings regardless of role.
</Note>

## Making Changes

Most settings auto-save as you make changes:

- Text fields save when you click away
- Toggles save immediately
- Dropdowns save when you select an option

Some settings require explicit save actions - look for **Save Changes** buttons.

## Impact of Changes

<Warning>
Changes to Organization Settings affect all locations and users in your organization. Make changes carefully and communicate with your team.
</Warning>

### Immediate Effects

These changes take effect right away:
- Role hierarchy changes
- User additions/removals
- Rating criteria updates

### Gradual Effects

These changes affect new data but not historical data:
- Position additions
- Infraction type changes
- Threshold adjustments

## Best Practices

<Steps>
  <Step title="Plan Changes">
    Before making significant changes, discuss with your leadership team.
  </Step>
  
  <Step title="Test First">
    If possible, test changes with a small group before rolling out organization-wide.
  </Step>
  
  <Step title="Communicate">
    Let your team know about changes that affect how they use Levelset.
  </Step>
  
  <Step title="Document">
    Keep notes on what you changed and why for future reference.
  </Step>
</Steps>
